# VEDA Multi-Sensor TCP Control System 실행 과정

본 문서는 VEDA Multi-Sensor TCP Control System의 빌드, 배포, 실행 및 각 기능별 시연 과정을 단계별로 정리한 가이드입니다. 실제 동작 결과(터미널 출력, 물리 작동 내용)를 함께 명시하여 교수의 검증 편의성을 돕습니다.

---

## 1. 실행 전 준비

### 대상 환경
* **서버 장치**: Raspberry Pi 4 (AArch64)
* **클라이언트**: Ubuntu Linux 또는 WSL Ubuntu (x86_64)
* **서버 포트**: TCP `8000`
* **배포 경로**: `/home/sunbi/prj`

### 필요 도구
* `make`
* `gcc`
* `aarch64-linux-gnu-gcc` (크로스 컴파일러)
* **Raspberry Pi sysroot**: `~/rpi-sysroot` (wiringPi 헤더 및 라이브러리가 내부에 포함되어 있어야 함)

### 하드웨어 연결 (핀맵)
| 장치 | wPi Pin | BCM GPIO | Physical Pin | 비고 |
|------|---------|----------|--------------|------|
| LED | 1 | 18 | 12 | PWM, 220Ω 저항 연결 |
| Buzzer | 2 | 27 | 13 | softTone 제어 |
| CDS SDA | 8 | 2 | 3 | PCF8591 ADC, I2C `0x48` |
| CDS SCL | 9 | 3 | 5 | PCF8591 ADC, I2C `0x48` |
| CDS Digital | 0 | 17 | 11 | Pull-up, 10kΩ |
| 7-Segment A | 21 | 5 | 29 | Common Anode |
| 7-Segment B | 22 | 6 | 31 | Common Anode |
| 7-Segment C | 23 | 13 | 33 | Common Anode |
| 7-Segment D | 24 | 19 | 35 | Common Anode |
| 7-Segment E | 25 | 26 | 37 | Common Anode |
| 7-Segment F | 27 | 16 | 36 | Common Anode |
| 7-Segment G | 28 | 20 | 38 | Common Anode |
| 7-Segment DP| 29 | 21 | 40 | Common Anode |

## 3. 빌드

빌드 환경에 따라 적합한 `Makefile`을 선택하여 사용해야 합니다.

* **개인 개발 환경 (sysroot 설정 적용)**: 기본 제공되는 `Makefile`을 그대로 사용합니다.
* **배포 및 일반 표준 환경 (sysroot 비적용)**: `Makefile.simple`을 복사하여 사용합니다.
  ```bash
  rm -f Makefile
  cp Makefile.simple Makefile
  ```

프로젝트 루트에서 `make clean && make` 명령을 실행하여 서버, 클라이언트 및 동적 라이브러리(.so)들을 한 번에 빌드합니다.

```bash
make clean && make
```

## 4. Raspberry Pi로 배포

`Makefile` 내에 선언된 `deploy` 타겟을 이용하여 한 번에 원격 빌드 및 업로드를 수행할 수 있습니다.

### 방법 A: Make 명령어로 자동 배포 (권장)
```bash
# 기본 변수(RPI_USR=sunbi, RPI_IP=192.168.0.100, RPI_DIR=/home/sunbi/prj)를 사용해 배포할 때
make deploy

# 특정 IP, 계정, 경로를 지정하여 배포할 때
make deploy RPI_IP=192.168.1.50 RPI_USR=pi RPI_DIR=/home/pi/myproject
```

### 방법 B: 수동 명령어로 배포
```bash
# 1. Raspberry Pi에 배포 폴더 생성
ssh <USR>@<RPI_IP> "mkdir -p <DIR>"

# 2. 서버 실행 파일 및 동적 라이브러리(.so) 전송
scp server lib*.so <USR>@<RPI_IP>:<DIR>

# 3. 웹 리소스 디렉토리 전송
scp -r resources <USR>@<RPI_IP>:<DIR>

# 4. 배포 완료 확인
ssh <USR>@<RPI_IP>
cd <DIR>
ls -F
```
**확인 대상**: `server*`, `libled.so`, `libbuzzor.so`, `libphotoresistor.so`, `libsegment.so`, `resources/`

## 5. 서버 실행

Raspberry Pi에서 서버를 구동합니다.

```bash
cd <DIR>
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH
```

### 디버그용 전경(Foreground) 실행
```bash
./server -f
```
**정상 실행 터미널 출력**:
```text
$ ./server -f
GPIO initialized successfully (using softPwm)
Server started on port 8000 with poll/multi-thread...
```

### 백그라운드 상시 구동 (데몬 모드)
```bash
./server
```
* 백그라운드 구동 후 프롬프트가 즉시 반환됩니다.

### 서버 종료 방법
* **전경 모드**: `Ctrl + C` 입력 시 메인 루프를 안전하게 종료하고 리소스를 닫은 후 `Shutdown successfully` 로그와 함께 종료됩니다.
* **데몬 모드**: `sudo killall server` 실행으로 프로세스를 중단합니다.

## 7. LED 제어 시연 및 실제 결과

### API 직접 호출 검증 및 기대 출력
1. **LED 켜기 (최대 밝기)**
   ```bash
   $ curl http://<RPI_IP>:8000/led/on
   OK
   ```
   * **물리 동작 결과**: 라즈베리파이에 연결된 LED가 최대 밝기로 점등됩니다.
   * **서버 터미널 로그**: `Raspberry Pi blink \n LED ON` 출력 확인.

2. **LED 밝기 조절 (PWM 300 지정)**
   ```bash
   $ curl "http://<RPI_IP>:8000/led/brightness?value=300"
   OK
   ```
   * **물리 동작 결과**: LED 밝기가 원래의 30% 수준으로 흐려집니다.
   * **서버 터미널 로그**: `LED brightness: 300 (mapped to softPwm 29)` 출력 확인.

3. **LED 끄기**
   ```bash
   $ curl http://<RPI_IP>:8000/led/off
   OK
   ```
   * **물리 동작 결과**: LED가 소등됩니다.
   * **서버 터미널 로그**: `LED OFF` 출력 확인.

## 8. 부저 제어 시연 및 실제 결과

### API 직접 호출 검증 및 기대 출력
1. **부저 계이름 '도' 연주**
   ```bash
   $ curl http://<RPI_IP>:8000/buzz/do
   OK
   ```
   * **물리 동작 결과**: 부저가 '도'(262Hz) 주파수로 0.5초간 짧게 소리 낸 후 자동으로 꺼집니다.
   * **서버 터미널 로그**: `Buzzer scale control: do \n Playing note do at 262 Hz for 500ms` 출력 확인.

2. **FEIN 알림음 재생**
   ```bash
   $ curl http://<RPI_IP>:8000/buzz/fein
   OK
   ```
   * **물리 동작 결과**: 특유의 빠른 알림 경보 멜로디가 부저에서 연속 재생됩니다.
   * **서버 터미널 로그**: `Playing Fein Alert Melody` 출력 확인.

3. **부저 수동 끄기**
   ```bash
   $ curl http://<RPI_IP>:8000/buzz/off
   OK
   ```
   * **물리 동작 결과**: 재생 중인 모든 부저 소리가 즉시 중단됩니다.
   * **서버 터미널 로그**: `Buzzer OFF` 출력 확인.

## 9. 조도센서 시연 및 실제 결과

### API 직접 호출 검증 및 기대 출력
1. **아날로그 조도 ADC 값 조회**
   ```bash
   $ curl http://<RPI_IP>:8000/pr
   148
   ```
   * **동작 결과**: PCF8591 I2C ADC 칩을 거쳐 읽어낸 조도 센서값(`0~255` 사이 정수)이 응답 본문으로 출력됩니다. 손으로 가릴 시 값이 증가(`200` 이상)하는 것을 확인할 수 있습니다.

2. **디지털 조도 값 조회**
   ```bash
   $ curl http://<RPI_IP>:8000/pr/digital
   1
   ```
   * **동작 결과**: 현재 주변이 밝으면 `1`, 가려서 어두워지면 `0`의 디지털 신호값이 출력됩니다.

3. **자동 조도 제어 켜기**
   ```bash
   $ curl http://<RPI_IP>:8000/pr/auto/start
   OK
   ```
   * **물리 동작 결과**: 백그라운드 자동 조도 제어 스레드가 1초마다 동작하여, 조도 센서를 가려서 어둡게 만들면(디지털 값 `0`) LED가 자동으로 점등되고, 손을 때서 밝아지면(디지털 값 `1`) LED가 즉시 자동으로 꺼지는 자동 제어가 동작합니다.

4. **자동 조도 제어 끄기**
   ```bash
   $ curl http://<RPI_IP>:8000/pr/auto/stop
   OK
   ```
   * **물리 동작 결과**: 자동 LED 제어가 종료되고 LED 상태가 수동 제어 모드로 돌아갑니다.

## 10. 7세그먼트 시연 및 실제 결과

### API 직접 호출 검증 및 기대 출력
1. **커스텀 카운트다운 시작 (5부터 시작)**
   ```bash
   $ curl http://<RPI_IP>:8000/segment/5
   OK
   ```
   * **물리 동작 결과**: 7세그먼트 디스플레이에 `5 -> 4 -> 3 -> 2 -> 1 -> 0` 순서로 1초 주기로 감소하며 숫자가 표시됩니다. `0`이 되는 즉시 LED가 켜지고 부저 Fein 알림 멜로디가 자동으로 1회 연주됩니다.
   * **서버 터미널 로그**:
     ```text
     Segment command: 5
     Starting Countdown from 5 to 0...
     Segment: 5
     Segment: 4
     Segment: 3
     Segment: 2
     Segment: 1
     Segment: 0
     Digit 0, LED ON
     Playing Fein Alert Melody
     ```

2. **카운트다운 도중 강제 중단**
   ```bash
   $ curl http://<RPI_IP>:8000/segment/start  # 9부터 감소 시작
   # (카운트다운 도중 다른 탭에서 아래 요청 호출)
   $ curl http://<RPI_IP>:8000/segment/off
   OK
   ```
   * **물리 동작 결과**: 9, 8, 7... 감소 중에 즉시 7세그먼트 표시와 LED가 소등되며 카운트다운 루프가 안전하게 중지됩니다.
   * **서버 터미널 로그**: `Countdown cancelled!` 출력 확인.

## 11. CLI 클라이언트 시연 및 실제 결과

호스트 PC에서 다음 명령어로 서버에 접속합니다.

```bash
$ ./client <RPI_IP>
```

### 실행 UI 및 메뉴 렌더링 결과
접속 성공 시 다음 메뉴가 터미널에 상시 루프 상태로 표출됩니다.
```text
1 LED  2 Buzzer  3 7-Segment  4 Photoresistor  5 Quit
Select menu: 
```

* **1 LED 선택 후 `ON` 입력**: `HTTP/1.1 200 OK`를 받아 물리 LED 점등 및 메뉴 재출력 확인. `9999` 같은 범위 외 값 입력 시 `Invalid brightness` 예외 로그가 클라이언트에 출력되고 원상태로 복귀합니다.
* **3 7-Segment 선택 후 `start` 입력**: 7세그먼트에 카운트다운이 돌아가고 동시에 클라이언트 터미널 메뉴로 즉시 복귀하여 다중 비동기 처리가 됨을 보います.
* **4 Photoresistor 선택 후 `1` 입력 (아날로그 조회)**: `Current ADC value: 135` 형태의 실제 아날로그 조도 수치가 터미널에 인쇄되는지 확인합니다.

## 12. 스레드 모니터링 시연 및 실제 결과

### API 호출 검증 및 기대 출력
1. **아무 제어 요청도 수행하지 않고 있을 때**
   ```bash
   $ curl http://<RPI_IP>:8000/api/threads
   []
   ```
2. **7세그먼트 카운트다운 동작 중일 때**
   ```bash
   $ curl http://<RPI_IP>:8000/segment/9 &
   $ curl http://<RPI_IP>:8000/api/threads
   [{"tid":139694464190208,"ip":"192.168.0.15","cmd":"segment/9","uptime":3}]
   ```
   * **동작 결과**: 현재 백그라운드 동기 지연 처리를 수행하는 세그먼트 카운트다운 쓰레드가 활성화 정보와 기동 시간(Uptime) 정보를 JSON 형태로 명확히 노출하는 것을 볼 수 있습니다. (웹 대시보드 하단의 테이블에도 동일한 정보가 1.5초 새로고침 간격에 맞추어 표 형태로 실시간 출력됩니다.)
