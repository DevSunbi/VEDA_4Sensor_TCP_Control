# 🎛️ VEDA Multi-Sensor TCP Control System

Raspberry Pi 4를 기반으로 동작하는 **TCP 멀티스레드 IoT 장치 통합 제어 시스템**입니다.  
웹 대시보드(Web Browser) 및 대화형 CLI 클라이언트(Client)를 통해 원격지에서 라즈베리 파이에 탑재된 LED, 부저, 7세그먼트, 조도센서를 실시간 제어하고 상태를 모니터링할 수 있습니다.

본 프로젝트는 멀티태스킹 임베디드 환경에서의 네트워크 데이터 병렬 멀티플렉싱, 하드웨어 장치 드라이버의 동적 로딩, 스레드 안전성 확보를 위한 뮤텍스 설계, 배포 환경 최적화를 위한 데몬화 기법을 직접 설계하고 구현하였습니다.

## 💡 주요 기능 및 구현 특징

### 1. 다중 입출력 멀티플렉싱 및 비동기 처리
* **`poll()` 기반 이벤트 멀티플렉싱**: 다수의 소켓 연결을 효율적으로 감시하여 싱글 스레드 동시 연결 대기 한계를 예방합니다.
* **`pthread_detach` 비동기 스레딩**: 클라이언트 요청이 수락(`accept`)되면 분석 및 센서 제어용 비동기 스레드를 독립적으로 분리(`detach`)시켜 처리하며, 메인 루프는 다음 소켓 입력을 즉시 수용할 수 있는 고반응성 구조를 가집니다.

### 2. 하드웨어 드라이버 동적 공유 라이브러리 연동
* **런타임 심볼 해석 (`dlopen` / `dlsym`)**: 하드웨어 제어 기능 모듈을 공유 라이브러리(`.so`)로 구축하고, 런타임에 동적으로 필요한 기능만 적재하여 실행합니다. 이를 통해 컴파일 의존성을 제거하고 부품 교체 및 드라이버 수정이 매우 직관적입니다.

### 3. 멀티스레드 하드웨어 제어 및 뮤텍스 정밀도 최적화
* **GPIO 경합 방지**: 여러 클라이언트의 API 요청 스레드가 동일한 센서 핀을 제어하려 할 때 데이터 무결성을 보장하도록 `pthread_mutex_t` 동기화 락 구조(`gpio_lock` / `gpio_unlock`)를 마련했습니다.
* **블로킹 딜레이 최적화**: 7세그먼트 카운트다운과 같이 소모성 시간(`delay(1000)`)을 가질 때 뮤텍스를 잡고 있으면 다른 스레드가 블로킹되는 문제를 피하기 위해, 하드웨어 핀 출력 동작 순간에만 뮤텍스를 잠그고 대기 중에는 즉각 해제하도록 락 스코프를 정밀 조율했습니다.

### 4. 세그먼트 비동기 카운트다운 타이머 & 캔슬 플래그
* **타이머 제어 이원화**: 카운트다운(`/segment/<digit>`)과 단발성 출력(`/segment/show/<digit>`)을 분리하여 웹 대시보드와 서버 간의 제어 경합을 최소화했습니다.
* **실시간 취소 감지**: 10초 내외의 카운트다운 도중 사용자가 `OFF` 요청을 주면 즉각 중단하도록 루프 회차마다 전역 `cancel_countdown_flag`를 확인합니다.
* **라이브러리 간 상호 작용**: 카운트다운이 종료되어 0이 되는 순간, `libsegment.so` 내부에서 `libbuzzor.so`를 즉석에서 동적 로드하여 알람 멜로디(`play_fein_style_alert`)를 연동 출력합니다.

### 5. 서버 주도 조도센서 백그라운드 자동 제어
* **`auto_pr_control_thread` 구동**: 클라이언트의 끊임없는 REST API 폴링 없이도 서버 백그라운드에서 주기적으로 센서 모듈의 디지털 출력을 읽습니다.
* **자율 피드백**: 주위 조도가 어두워지면(1) LED를 켜고, 밝아지면(0) 끄는 임베디드 자율 동작 모드를 구현했습니다.

### 6. CLI 클라이언트 안정성 및 시그널 핸들링
* **클라이언트 비정상 종료 예방**: 콘솔 클라이언트(`client.c`) 종료 시 소켓 누수와 서버 리소스 대기 상태를 안전하게 방지하기 위해 `SIGINT` 핸들러를 도입했습니다. 
* **종료 전 안전 조치**: Ctrl+C 감지 시, 동작 중일 수 있는 자동 조도 제어를 명시적으로 비활성화한 뒤 소켓을 `close()`하고 깨끗하게 프로세스를 퇴장시킵니다.
* **시그널 무시**: 임시 대기 상태로 빠져나갈 수 있는 `SIGTSTP` 및 `SIGQUIT`를 강제로 무시(`SIG_IGN`) 설정해 비정상 실행 흐름을 제어합니다.

### 7. 백그라운드 데몬 프로세스 기법
* **완벽한 백그라운드 영속성**: 이중 포크(`fork`)를 사용해 터미널과의 관계를 끊고 고유의 세션 ID(`setsid`)를 할당받아 배포 수준의 백그라운드 실행을 확보합니다.
* **실행 디렉토리 보호**: `/proc/self/exe`의 경로 분석을 통해 데몬이 항상 바이너리가 위치한 현재 디렉토리를 작업 경로로 사용하도록 안전을 유지합니다.
* **입출력 리다이렉션**: 디폴트 스트림(0, 1, 2)을 닫고 `/dev/null`로 연결해 로그 찌꺼기로 인한 비정상 프로세스 정지를 예방합니다. (`-f` 옵션 시 포그라운드 전환 가능)

| `/led.html` | GET | LED 밝기 및 ON/OFF 제어 페이지 |
| `/buzzer.html` | GET | 부저 가상 건반 제어 페이지 |
| `/segment.html` | GET | 7세그먼트 제어 및 타이머 페이지 |
| `/pr.html` | GET | 조도센서 실시간 모니터링 페이지 |

| `/led/off` | GET | LED 끄기 |

| `/buzz/<note>` | GET | 부저 음계 재생 (`do`~`do2`, 끄기는 `off` / 주파수 정수 / `fein` 또는 `alert` 멜로디 재생) |
| `/segment/start` | GET | 9부터 0까지 기본 카운트다운 시작 및 종료 시 부저 멜로디 호출 |
| `/segment/<digit>` | GET | 지정한 `<digit>` 숫자(1~9)부터 0까지 카운트다운 시작 |
| `/segment/show/<digit>` | GET | 지정한 `<digit>` 숫자를 단발성으로 7세그먼트에 출력 |
| `/segment/off` | GET | 카운트다운 루프 즉시 종료 및 세그먼트/LED 소등 |
| `/pr` | GET | 조도센서 PCF8591 I2C 아날로그 ADC 값 조회 (0~255) |
| `/pr/digital` | GET | 조도센서 GPIO 디지털 상태 조회 (0: 밝음 / 1: 어두움) |
| `/pr/auto/start` | GET | 서버 내 백그라운드 조도센서 자동 LED 제어 활성화 |
| `/pr/auto/stop` | GET | 서버 내 백그라운드 조도센서 자동 제어 비활성화 및 스레드 종료 |

## 🔨 빌드 및 실행 방법

### 1. 사전 요구사항
* **호스트 PC**: 타겟 빌드를 위한 `aarch64-linux-gnu-gcc` 크로스 컴파일러 설치 및 라즈베리 파이 OS용 `~/rpi-sysroot` 준비가 완료되어 있어야 합니다.

### 2. 빌드 실행 (Host PC)
프로젝트 루트 디렉토리에서 메이크파일을 실행합니다.
```bash
# 빌드 실행
make

# 리빌드 실행
make clean && make
```
* **결과물**: `server` (타겟 실행 파일), `client` (호스트용 CLI), `lib*.so` (동적 모듈 라이브러리들)

### 3. 파일 배포 (Host PC → Raspberry Pi)
빌드된 빌드 바이너리와 HTML 리소스 폴더를 타겟 장치로 동기화합니다.
```bash
# 바이너리 파일 전송
scp server lib*.so sunbi@<RPI_IP>:/home/sunbi/projectHome/

# 리소스 폴더 재귀 전송
scp -r resources/ sunbi@<RPI_IP>:/home/sunbi/projectHome/
```

### 4. 서버 구동 및 제어

**라즈베리 파이 (서버 실행):**
```bash
cd /home/sunbi/projectHome
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH

# 1. 백그라운드 데몬 프로세스로 상시 구동
./server

# 2. 디버깅 모드(포그라운드 로그 출력)로 구동
./server -f
```

**호스트 PC (제어 및 클라이언트 실행):**
* **CLI 원격 제어 콘솔**:
  ```bash
  ./client <RPI_IP>
  ```
* **웹 대시보드 원격 접속**:
  웹 브라우저 주소창에 `http://<RPI_IP>:8000/` 접속
