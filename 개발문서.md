# VEDA Multi-Sensor TCP Control System 개발문서

## 1. 프로젝트 개요

본 프로젝트는 TCP 통신을 이용하여 Ubuntu Linux 클라이언트 또는 웹 브라우저에서 Raspberry Pi 4에 연결된 장치를 원격 제어하는 리눅스 시스템 프로그래밍 실습 프로젝트이다.

평가 과제에서 요구하는 대상 장치는 LED, 부저, 조도센서, 7세그먼트이며, 각 장치 제어 기능은 공유 라이브러리(`.so`)로 분리하였다. 서버는 실행 중 `dlopen()`과 `dlsym()`으로 장치별 함수를 동적으로 로딩하므로 특정 장치 기능을 수정하거나 업그레이드할 때 해당 라이브러리만 재빌드 및 재배포할 수 있다.

## 2. 요구사항 대응

| 요구사항 | 구현 파일 | 구현 내용 |
|----------|-----------|-----------|
| TCP 원격 장치 제어 | `src/main.c`, `src/client.c` | 서버는 TCP 8000 포트에서 요청을 받고, CLI 클라이언트는 HTTP 형식의 GET 요청을 전송 |
| 서버: Raspberry Pi 4 | `src/main.c`, `lib/*.c` | wiringPi 기반 GPIO, PWM, I2C 제어 |
| 클라이언트: Ubuntu Linux | `src/client.c` | 호스트 PC용 `gcc`로 빌드되는 CLI 메뉴 제공 |
| 멀티 프로세스 또는 스레드 | `src/main.c` | `poll()`로 연결을 감시하고 요청별 detached pthread 생성 |
| 장치 공유 라이브러리 | `lib/led.c`, `lib/buzzor.c`, `lib/photoresistor.c`, `lib/segment.c` | `libled.so`, `libbuzzor.so`, `libphotoresistor.so`, `libsegment.so` 생성 |
| 클라이언트 시그널 처리 | `src/client.c` | `SIGINT`에서 자동 제어 중지 및 소켓 정리, `SIGTSTP`/`SIGQUIT` 무시 |
| 서버 데몬 구성 | `src/main.c` | 기본 실행 시 이중 fork 데몬화, `-f`로 전경 실행 |
| 빌드 자동화 | `Makefile` | `make`, `make clean` 제공 |

## 4. 시스템 구조

```text
Ubuntu Linux CLI client        Web Browser
          |                        |
          +---------- TCP 8000 ----+
                                   |
                              src/main.c
                         poll + pthread server
                                   |
       +---------------------------+---------------------------+
       |                           |                           |
   libled.so                 libbuzzor.so              libsegment.so
       |                           |                           |
       +------------------ lib/rpi_common.c ------------------+
                                   |
                         libphotoresistor.so
                                   |
                         Raspberry Pi 4 GPIO/I2C
```

서버는 정적 HTML 파일을 `resources/`에서 제공하며, 장치 제어 요청은 `/led`, `/buzz`, `/segment`, `/pr` 계열 API로 라우팅한다.

## 5. 세부 구현 내용

### 5.1 서버

`src/main.c`는 TCP 서버의 중심 모듈이다.

| 항목 | 구현 |
|------|------|
| 포트 | `PORT 8000` |
| 연결 처리 | listening socket을 `poll()`로 감시 |
| 요청 처리 | 클라이언트 fd를 detached pthread에 넘겨 병렬 처리 |
| 동적 로딩 | `dlopen("./libled.so")`, `dlsym("led")` 방식 |
| 정적 파일 | `/`는 `resources/index.html`, 그 외 HTML은 `resources/<filename>`에서 제공 |
| 데몬화 | 기본 실행 시 이중 fork, `setsid()`, 표준 입출력 `/dev/null` 리다이렉션 |
| 디버그 실행 | `./server -f` 또는 `./server --foreground` |

데몬 실행 시 `/proc/self/exe`로 실제 실행 파일 경로를 찾고 해당 디렉토리로 이동한다. 이 때문에 `server`, `lib*.so`, `resources/`를 같은 배포 디렉토리에 두면 상대 경로로 라이브러리와 HTML을 찾을 수 있다.

### 5.2 클라이언트

`src/client.c`는 Ubuntu Linux에서 실행하는 CLI 클라이언트이다.

| 메뉴 | 기능 |
|------|------|
| 1 LED | ON, MID, OFF, 0~1024 밝기 값 입력 |
| 2 Buzzer | `do`, `re`, `mi`, `fa`, `sol`, `la`, `si`, `do2`, `off` 입력 |
| 3 7-Segment | `start`, `off` 명령 입력 |
| 4 Photoresistor | 아날로그 조회, 디지털 조회, 자동 LED 제어 시작/중지 |
| 5 Quit | 자동 제어 중지 요청 후 종료 |

`SIGINT` 발생 시 `pr/auto/stop` 요청을 보내고 열린 소켓을 닫는다. `SIGTSTP`, `SIGQUIT`은 무시하여 의도하지 않은 정지나 종료를 줄였다.

### 5.3 LED

`lib/led.c`는 LED 제어를 담당한다.

| 명령 | 동작 |
|------|------|
| `ON` | `pwmWrite(LED_PIN, 1024)` |
| `MID` | `pwmWrite(LED_PIN, 512)` |
| `OFF` | `pwmWrite(LED_PIN, 0)` 및 카운트다운 취소 플래그 설정 |
| `0~1024` | PWM 밝기 직접 제어 |

LED는 wiringPi 기준 1번 핀, BCM 18번 핀을 사용하며 PWM 출력으로 설정한다.

### 5.4 부저

`lib/buzzor.c`는 `softTone` 기반 부저 제어를 담당한다.

| 명령 | 동작 |
|------|------|
| `do`~`do2` | 계이름 주파수 재생 |
| `1`~`8` | 숫자 음계 재생 |
| 직접 주파수 | 입력 숫자를 Hz로 해석해 500ms 재생 |
| `OFF`, `0` | 부저 정지 |
| `fein`, `alert` | 알림 멜로디 재생 |

### 5.5 조도센서

`lib/photoresistor.c`는 PCF8591 I2C ADC와 디지털 조도 입력을 함께 지원한다.

| 함수 | 용도 |
|------|------|
| `get_cds_value()` | PCF8591 `0x48`에서 AIN0 아날로그 값을 읽음 |
| `get_pr_value()` | wiringPi 0번 핀의 디지털 값을 읽음 |
| `cdsControl()` | 독립 실행형 무한 루프 제어 함수. 서버에서는 종료성 문제 때문에 직접 사용하지 않음 |

현재 서버 자동 제어는 `get_pr_value()`를 1초마다 호출한다. 디지털 값이 `LOW(0)`이면 어두움으로 판단해 LED를 켜고, `HIGH(1)`이면 밝음으로 판단해 LED를 끈다.

### 5.6 7세그먼트

`lib/segment.c`는 Common Anode 7세그먼트를 제어한다.

| 명령 | 동작 |
|------|------|
| `start` | 9부터 0까지 1초 간격 카운트다운 |
| `0~9` | 지정 숫자부터 0까지 카운트다운 |
| `show/<0~9>` | 지정 숫자 단발 표시 |
| `OFF` | 표시 제거, LED OFF, 카운트다운 취소 |

카운트다운 중에는 `delay(1000)` 구간에서 GPIO 뮤텍스를 풀어 다른 LED/부저 요청이 오래 막히지 않도록 했다. 0에 도달하면 LED를 켜고 `libbuzzor.so`에서 `play_fein_style_alert`를 동적으로 찾아 알림음을 재생한다.

## 6. 웹 대시보드

`resources/` 디렉토리는 브라우저용 HTML UI를 제공한다.

| 파일 | 기능 |
|------|------|
| `index.html` | 장치별 페이지 진입 |
| `led.html` | LED ON/MID/OFF 및 밝기 슬라이더 |
| `buzzer.html` | 계이름/알림음 부저 제어 |
| `segment.html` | 7세그먼트 단일 표시, 기본/커스텀 카운트다운, OFF |
| `pr.html` | 조도센서 아날로그/디지털 값 조회, 자동 LED 제어 |

## 7. 빌드 및 배포 구조

`Makefile`은 서버와 공유 라이브러리를 Raspberry Pi용 AArch64 바이너리로 빌드하고, CLI 클라이언트는 호스트용 `gcc`로 빌드한다.

| 결과물 | 대상 |
|--------|------|
| `server` | Raspberry Pi |
| `client` | Ubuntu Linux host |
| `libled.so` | Raspberry Pi |
| `libbuzzor.so` | Raspberry Pi |
| `libphotoresistor.so` | Raspberry Pi |
| `libsegment.so` | Raspberry Pi |
| `resources/` | Raspberry Pi |

배포 기준 경로는 `/home/sunbi/prj`이다.

## 8. 문제점 및 보완 사항

GitHub Issues 기준으로 프로젝트의 문제점을 추적했다. 2026-06-04 KST 기준 PR을 제외한 실제 이슈는 총 20건이며, Open 2건, Closed 18건이다.

| 구분 | 내용 | 상태 |
|------|------|------|
| 정적 리소스 경로 불일치 | HTML 파일 경로가 실행 환경과 달라 `index.html`을 찾지 못함 | 해결 |
| LED 핀 매핑 오류 | 실제 배선은 wPi 1/BCM 18인데 코드 정의가 다름 | 해결 |
| 동적 모듈 상태 불일치 | 모듈별 전역 상태가 `dlopen`/`dlclose` 사이에서 유지되지 않음 | 해결 |
| 7세그먼트 취소 레이스 | OFF 요청 후에도 카운트다운이 끝까지 진행됨 | 해결 |
| `/pr`와 `/pr.html` 라우팅 충돌 | 접두사 비교로 HTML 요청이 API로 오인됨 | 해결 |
| LED PWM 전환 후 digitalWrite 충돌 | PWM 핀에 `digitalWrite`를 사용해 LED 상태가 불안정 | 해결 |
| 데몬 경로 문제 | 데몬 전환 후 작업 디렉토리가 잘못되어 리소스를 찾지 못함 | 해결 |
| 부저 라이브러리 오타 | `gpio_unlck()` 오타로 런타임 오류 가능 | 해결 |
| 클라이언트 수신 안정성 | 단발 `recv()`로 응답 일부가 잘릴 수 있음 | 해결 |
| 클라이언트 입력 검증 | `atoi()`가 잘못된 문자열을 0으로 처리 | 해결 |
| 조도센서 내부 루프 종료성 | `cdsControl()`이 무한 루프라 서버 스레드로 직접 사용하기 어려움 | 보완 필요 |
| 카운트다운 시간 지터 | 뮤텍스 경합과 `delay(1000)` 기반 대기로 정확한 1초 간격이 밀릴 수 있음 | 보완 필요 |
| 클라이언트 연결 실패 UX | 서버 접속 실패 시 재시도 백오프가 없음 | 보완 필요 |
